# Последнее слово: dashboard

Статичное readonly-зеркало материалов сайта “Последнее слово” для GitHub Pages.

Текущая публичная рабочая база по умолчанию русскоязычная: `/words/`, отчеты по именам,
тегам и проверка отсутствующей даты произнесения скрывают англоязычные записи. Английский
поток и связи переводов выделены в отдельные будущие задачи.

Репозиторий содержит только то, что нужно для сборки публичного сайта:

- Astro-приложение в `src/`
- контентные Markdown-файлы в `src/content/last-words/`
- минимальные скрипты импорта в `scripts/`
- зависимости и конфигурацию сборки

Локальные выгрузки WordPress не входят в публичный репозиторий.

## Команды

```sh
npm install
npm run dev
npm test
npm run test:integrity
npm run integrity:update
npm run validate:content
npm run build
npm run preview
npm run fetch:wp
npm run import:wp
```

## Обновление базы

GitHub Actions обновляет контент автоматически:

- ежедневно: incremental sync по `modified_after`
- раз в неделю: full sync с удалением локальных Markdown-файлов, которых больше нет среди опубликованных WordPress-постов
- вручную: workflow `Update WordPress content` можно запустить в режиме `incremental` или `full`

Сырые WordPress JSON сохраняются только во временный ignored-каталог `data/` и не коммитятся.

Дата последней синхронизации хранится в `src/data/wpExportMeta.json`. Главная страница считает
“Последние добавления за неделю” как 7 полных дней перед датой этой синхронизации.
Инкрементальный журнал действий импорта хранится в `src/data/wpImportLog.jsonl`: одна строка
JSON на каждое созданное, обновленное или удаленное слово.

Главная показывает канонические счетчики текущей базы: 503 слова, 450 русскоязычных и
53 англоязычных записи. Основные русскоязычные представления временно отсекают записи с английским
`sourceUrl`, пока не введены окончательные поля переводов и языка оригинала.

Для GitHub Pages используется base path `/fs_dashboard`, он задан в `astro.config.mjs`.

`npm run build` автоматически запускает `validate:content`. Проверка падает, если в
`src/content/last-words/` есть отсутствующие, некорректные или повторяющиеся `wordpressId`.

`npm test` проверяет snapshot статистики главной страницы: общее количество слов, разбивку по
языкам, количество людей и список стран. Ожидаемые значения лежат в
`scripts/statistics-expectations.json`; после осознанного обновления базы их нужно обновить.

`npm run test:integrity` защищает тексты от массовой порчи: сравнивает длину, количество слов
и SHA-256 очищенного текста с `scripts/content-integrity-baseline.json`. Проверка падает, если
тексты стали пустыми, резко сократились или массово изменились. В workflow обновления контента
эта проверка запускается после импорта и до сборки; baseline обновляется только после успешной
проверки командой `npm run integrity:update`.
