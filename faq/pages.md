# Страницы сайта

Этот файл описывает текущие и планируемые страницы дашборда. Его можно править вручную,
когда меняется структура, назначение страниц или состав блоков.

## Главная

URL: `/`

Файл: `src/pages/index.astro`

Источник данных:

- Astro Content Collection `last-words`.

Назначение:

- дать быстрый обзор зеркала;
- показать, что это readonly-дашборд;
- вывести базовую статистику;
- дать вход в базу слов и описание проекта.

Сейчас на странице:

- заголовок и короткое описание;
- кнопки “Открыть базу слов” и “Как это устроено”;
- счетчики: слов, человек, стран;
- блок “Последние добавления за неделю”, рассчитанный относительно самой свежей записи в
  коллекции.

Возможные изменения:

- добавить ссылку на отчеты;
- добавить блок “последние проблемы в данных”;
- добавить статус последней синхронизации с WordPress.

## База слов

URL: `/words/`

Файл: `src/pages/words/index.astro`

Источник данных:

- Astro Content Collection `last-words`.

Назначение:

- показать все последние слова в одном списке;
- дать быстрый обзор базы;
- стать основной рабочей страницей для просмотра записей.

Сейчас на странице:

- заголовок и описание;
- поиск, фильтр и сортировка с пометкой, что они пока не работают стабильно;
- количество видимых записей;
- карточки последних слов.

Карточка сейчас показывает:

- заголовок;
- описание дела;
- теги;
- страну и эпоху в формате `Россия: 2020-е`;
- время чтения в формате `Время чтения: 5 минут`.

Возможные изменения:

- довести поиск, фильтры и сортировку до рабочего состояния;
- добавить фильтр по тегам;
- добавить фильтр по стране;
- добавить фильтр по эпохе;
- добавить фильтр “есть фото / нет фото”;
- добавить фильтр “есть перевод / нет перевода”;
- перейти от карточек к более плотной таблице.

## Страница последнего слова

URL: `/words/[slug]/`

Файл: `src/pages/words/[slug].astro`

Источник данных:

- Astro Content Collection `last-words`.

Назначение:

- показать один текст последнего слова;
- показать основные метаданные;
- дать ссылку на оригинал в WordPress;
- позже показать переводы и связанные слова.

Сейчас на странице:

- заголовок;
- описание человека;
- описание дела;
- кнопка назад к базе;
- ссылка на источник;
- полный HTML-текст из WordPress;
- правая колонка с фото, если оно есть;
- под фото: страна и эпоха, время чтения, теги.

Что принципиально не показываем сейчас:

- дату публикации WordPress-поста.

Возможные изменения:

- вывести язык оригинала;
- вывести переключатель переводов;
- вывести связанные последние слова;
- отдельно показать суд, город и дату произнесения, если эти данные появятся структурно;
- добавить блок источников и ссылок на подробности;
- очистить WordPress HTML от лишних классов и пустых параграфов.

## Отчет: неполные поля

URL: `/reports/missing-fields/`

Файл: `src/pages/reports/missing-fields.astro`

Источник данных:

- Astro Content Collection `last-words`.

Назначение:

- показать записи, которым не хватает важных импортируемых данных;
- помочь команде быстро находить карточки для ручной проверки;
- проверить, что на основе frontmatter можно строить аналитические страницы.

Сейчас на странице:

- показываются только группы, в которых есть хотя бы одна проблемная запись;
- проверки: без фото, без описания человека, без тегов, без эпохи;
- в каждой карточке есть ссылка на локальную страницу и кнопка “Посмотреть источник”.

Что пока не проверяем:

- суд;
- город;
- дату произнесения.

Этих данных пока нет как отдельных структурных полей, поэтому их отсутствие не считается
ошибкой импорта.

Возможные изменения:

- добавить “без языка оригинала”, когда язык перестанет быть дефолтом;
- добавить “без настроенной группы переводов”;
- добавить “без источника”;
- добавить “без описания дела”;
- добавить severity: критично / желательно / косметика;
- добавить экспорт отчета в JSON или CSV.
