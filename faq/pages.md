# Страницы сайта

Этот файл описывает текущие и планируемые страницы дашборда. Его можно править вручную,
когда меняется структура, назначение страниц или состав блоков.

## Главная

URL: `/`

Файл: `src/pages/index.astro`

Источник данных:

- Astro Content Collection `last-words`.
- `src/data/wpExportMeta.json` для даты последней синхронизации.

Назначение:

- дать быстрый обзор зеркала;
- показать, что это readonly-дашборд;
- вывести базовую статистику;
- дать вход в базу слов и описание проекта.

Сейчас на странице:

- заголовок и короткое описание;
- кнопки “Открыть базу слов” и “Как это устроено”;
- счетчики: канонических слов, уникальных русскоязычных людей и стран;
- разбивка канонических слов по языку записи: 450 на русском и 53 на английском;
- счетчик стран считается только по русскоязычной рабочей выборке и исключает значения, которые
  являются регионами или служебными рубриками;
- под счетчиком стран мелко перечисляются страны, которые попали в счетчик;
- блок “Последние добавления за неделю”: 7 полных календарных дней перед датой
  синхронизации. Например, если база синхронизирована 21 мая, период будет 14-20 мая
  включительно.

Возможные изменения:

- добавить ссылку на отчеты;
- добавить блок “последние проблемы в данных”;
- уточнить, какие типы дат должны участвовать в главной сводке: публикация, обновление или
  дата произнесения последнего слова.

## Отчет: неполные поля

URL: `/reports/missing-fields/`

Файл: `src/pages/reports/missing-fields.astro`

Источник данных:

- Astro Content Collection `last-words`.

Назначение:

- показать записи, которым не хватает важных импортируемых данных;
- помочь команде быстро находить карточки для ручной проверки;
- проверить, что на основе frontmatter можно строить аналитические страницы.

Сейчас на странице:

- показываются только группы, в которых есть хотя бы одна проблемная запись;
- проверки: без фото, без описания человека, без тегов, без эпохи, без даты произнесения;
- проверка отсутствия даты произнесения должна применяться только к русскоязычным записям,
  потому что текущий парсер даты ориентирован на русский нижний служебный блок;
- для проверки даты используется русскоязычная рабочая выборка: `language === "ru"` и
  `sourceUrl` не содержит `/en/`;
- в каждой карточке есть ссылка на локальную страницу и кнопка “Посмотреть источник”.

Что пока не проверяем:

- суд;
- город;
- место произнесения как отдельное структурное поле;
- источник как отдельное структурное поле;
- ссылку на описание дела как отдельное структурное поле;
- автора фото как отдельное структурное поле.

Дата произнесения хранится в `statementDate`. Суда, города, места произнесения, источника,
ссылки на описание дела и автора фото пока нет как отдельных структурных полей.

Возможные изменения:

- добавить “без языка оригинала”, когда язык перестанет быть дефолтом;
- добавить “без настроенной группы переводов”;
- добавить “без источника”;
- добавить “без описания дела”;
- добавить отдельную страницу ошибок по разным параметрам данных:
  - английские значения в русскоязычных словах и русские значения в англоязычных словах;
  - некорректно заполненные поля: дата произнесения, время чтения, язык, страна, эпоха;
  - служебные рубрики или регионы в полях, где ожидается страна;
- добавить severity: критично / желательно / косметика;
- добавить экспорт отчета в JSON или CSV.

## Отчет: языки оригинала

URL: `/reports/original-languages/`

Файл: `src/pages/reports/original-languages.astro`

Источник данных:

- Astro Content Collection `last-words`;
- вычисляемый разбор тела русскоязычной записи.

Назначение:

- собрать русскоязычные записи, где текст указывает на другой язык оригинала;
- показать найденный язык оригинала и код языка;
- показать причины определения языка: например, явный заголовок в тексте или уже заполненное
  поле `originalLanguage`.

Сейчас на странице:

- проверяется только русскоязычная рабочая выборка;
- ищутся явные маркеры вроде “Оригинал последнего слова на кыргызском языке”, “Последнее
  слово в оригинале на азербайджанском языке”, “Последнее слово было произнесено на
  каракалпакском языке” и варианты с беларускай мовой;
- записи группируются по языку оригинала;
- в сводном блоке показывается количество записей, количество найденных языков и список этих
  языков;
- для каждой записи показывается строка-доказательство, по которой язык был определен;
- если разные причины указывают на разные языки, запись помечается как конфликтная.

Важно:

- отчет пока не меняет frontmatter и не проставляет `originalLanguage` в content-файлах;
- признак перевода вычисляется как `language !== originalLanguage`, когда структурное поле
  будет заполнено.

## Планируемая страница: поиск

URL: `/search/`

Файл: пока нет.

Источник данных:

- статичный поисковый индекс, собранный из `last-words`.
- индекс генерируется во время сборки и публикуется как обычный JSON-файл.

Назначение:

- full-text поиск по базе.

Что должно быть:

- поле поиска;
- результаты по имени, описанию дела, описанию человека, тегам и тексту последнего слова;
- счетчик найденных записей;
- объяснение совпадений: имя, описание дела, описание человека, тег или текст;
- быстрый переход к странице слова;
- фильтры по языку записи, стране, эпохе, тегу и наличию фото;
- корректная работа на GitHub Pages без сервера.

Минимальная первая версия:

- `src/pages/search-index.json.ts` генерирует индекс из Content Collection;
- `src/pages/search.astro` загружает индекс в браузере;
- поиск работает простой проверкой строк без внешней библиотеки;
- результат показывает имя, краткое описание, теги и ссылку на слово.

Возможное развитие:

- подключить MiniSearch или Fuse.js, если простой поиск станет медленным или недостаточно
  точным;
- добавить подсветку фрагментов текста;
- разделить индекс на метаданные и полный текст, если JSON станет слишком тяжелым.
