# Цель проекта

Сделать readonly-дашборд для команды “Последнего слова”: простой сайт, где можно быстро
читать, искать, сравнивать и анализировать последние слова без работы в сложной
WordPress-админке.

Основной сайт остается на WordPress. Редактирование, модерация и публикация новых
материалов продолжают жить там. Дашборд нужен как удобное зеркало и рабочий инструмент для
анализа.

Архитектура и стек описаны отдельно: `faq/architecture.md`. Дорожная карта вынесена в
`faq/roadmap.md`.

## Что должно получиться

На первом этапе сайт должен уметь:

- показывать список всех последних слов;
- открывать отдельную страницу каждого последнего слова;
- показывать базовые метаданные: имя, дело, суд, дата, место, теги, язык, источник;
- давать команде быстрый readonly-доступ к текстам;
- работать как статичный сайт на GitHub Pages.

Позже поверх зеркала можно добавлять:

- таблицу со всеми записями;
- поиск;
- фильтры и сортировку;
- статистику;
- страницы тегов, стран, эпох и языков;
- проверку орфографии и качества данных;
- отчеты по неполным или подозрительным карточкам;
- экспорт выбранных слов в нужном формате.

## Принципы продукта

- Дашборд не заменяет WordPress-админку.
- На опубликованном сайте все работает только на чтение.
- Интерфейс должен быть проще, быстрее и понятнее, чем админка.
- Первым делом нужно сделать надежное зеркало, а не все инструменты сразу.
- Новые функции должны помогать команде работать с текстами, а не усложнять процесс.

## Основные сценарии

Команда должна иметь возможность:

- быстро найти последнее слово по имени человека;
- открыть текст и прочитать его без лишней навигации;
- посмотреть источник и основные данные по делу;
- увидеть, какие слова относятся к одному делу или процессу;
- найти тексты по тегам, стране, периоду или языку;
- понять, какие записи неполные или требуют проверки;
- получить сводную статистику по базе.

## Страницы сайта

Минимальная версия:

- главная страница;
- список всех последних слов;
- отдельная страница последнего слова;
- страница “О проекте”;
- страница всех тегов.

На странице списка:

- запись в списке соответствует именно последнему слову, а не человеку;
- один человек может встречаться несколько раз;
- рядом полезно показывать дату или год;
- нужна сортировка по дате и по алфавиту;
- нужны базовые фильтры.

На странице последнего слова:

- фото, если есть;
- имя человека;
- информация о деле;
- суд, город, страна и дата;
- язык оригинала;
- кнопки переводов, если есть;
- полный текст последнего слова;
- ссылки на источник и дополнительные материалы;
- теги;
- связанные последние слова, если есть.

## Навигация и фильтры

Первые полезные фильтры:

- по тегам;
- по стране;
- по эпохе или периоду;
- по языку;
- по наличию перевода;
- по наличию фото;
- по полноте метаданных.

Отдельно стоит сделать страницы группировок:

- все теги;
- все страны;
- все эпохи;
- все языки.
