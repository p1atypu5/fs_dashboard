Эти поля можно импортировать напрямую.

## Медиа

У поста:

- `featured_media`: `0`;
- endpoint `wp/v2/media?parent=19397` вернул пустой массив.

Для этого конкретного поста фото через стандартные поля REST API не найдено. Для других
постов нужно проверить оба варианта:

- `featured_media` + `_embedded.wp:featuredmedia`;
- attachments через `wp/v2/media?parent=<postId>`.

## Что не удалось достать структурно

В этом ответе нет отдельных полей:

- суд;
- город;
- дата произнесения последнего слова;
- язык оригинала;
- связи с переводами;
- связи с другими последними словами;
- фото.

Часть этих данных может быть внутри текста, excerpt, Yoast description или других ACF-полей, но в текущем публичном REST API они не выглядят как отдельные структурные поля.

## Что можно делать дальше

1. Скачать 10-20 постов и сравнить структуру.
2. Проверить, всегда ли доступны `acf.opisanie_cheloveka`, `acf.opisanie_proczessa`,
   `acf.vremya_chteniya`.
3. Проверить посты с фотографиями.
4. Проверить посты с переводами, если они есть на сайте.
5. Решить, сохраняем ли `content.rendered` как HTML или конвертируем в Markdown.
6. Описать схему Astro Content Collection на основе реально доступных полей.
